# Data Contracts

This document defines the files that scripts expect and produce.

## Speech Corpus

Path: `artifacts/final_v14/all_speakers.csv`

Required columns:

- `speech_id`: stable speech identifier, for example `SP_00000`.
- `date`: Hansard sitting date.
- `speaker`: normalized speaker string.
- `text`: speech text.

Optional columns may be added by downstream scripts, but the four required columns must remain stable.

## Cluster Assignments

Path: `artifacts/final_v14/v11_cluster_assignments.csv`

Required columns:

- `speech_id`
- `date`
- `year`
- `speaker`
- `hdbscan_cluster`
- `kmeans_cluster`
- `agglomerative_cluster`

HDBSCAN noise is represented as `-1`.

## Macro-Topic Assignments

Path: `artifacts/final_v14/macro_topic_assignments.csv`

Required columns:

- `speech_id`
- `macro_topic`

Expected values include `Macro-Topic 0` through `Macro-Topic 29` and `Procedural Noise`.

## Keyword JSON

Path: `artifacts/final_v14/macro_topic_keywords.json`

Expected top-level keys:

- `count`
- `tfidf`

Each section maps a macro-topic label to an ordered list of keywords.

## Ground Truth

Path: `data/evaluation/ground_truth_300.csv`

Required columns:

- `annotation_id`
- `speech_id`
- `date`
- `speaker`
- `text`
- `label`
- `label_reason`
- `match_status`
- `is_evaluable`

`ground_truth_evaluable_273.csv` is the subset whose text matched the current v14 corpus and can be joined to cluster assignments.

## Evaluation Outputs

Generated by `python src/evaluation/run_evaluation.py`:

- `artifacts/evaluation/table_iii_metrics.csv`
- `artifacts/evaluation/topic_quality_metrics.csv`
- `artifacts/evaluation/evaluation_summary.md`

## Extraction Audit

Inputs:

- `data/extraction_audit/reference_outputs/<sample_id>.txt`
- `data/extraction_audit/gemini_outputs/<sample_id>.txt`
- `data/extraction_audit/tesseract_outputs/<sample_id>.txt`

Generated output:

- `data/extraction_audit/extraction_audit_results.csv`

Required output columns:

- `sample_id`
- `system`
- `candidate_file`
- `reference_file`
- `word_accuracy`
- `speaker_boundary_f1`
- `layout_order_similarity`
- `missing_candidate`
